📊 TRON 能量租赁平台 - 需求与进度报告

🎯 项目定位
类似 thcal 的能量租赁平台，但我们做得更强：
• API 自动化 - 给开发者/企业用
• TG 机器人 - 普通用户用

参考：https://www.thcal.org/home

💰 业务模式
我们自己囤 TRX（比如 1 万个），用户要能量时：

1️⃣ 我们质押 TRX → 获得能量
2️⃣ 委托能量给用户（按小时）
3️⃣ 用户付钱，我们赚差价
4️⃣ 到期后能量自动回收，TRX 可以再用

✅ 好处：不依赖供应商，利润全是自己的，响应快

🚀 两大核心功能

1️⃣ API 自动化
   给开发者/企业用，可以集成到他们系统里自动买能量

2️⃣ TG 机器人
   普通用户用，发个命令就能买能量、查余额、自动续费

🔧 技术栈
Node.js + MySQL + TronWeb（直接对接 TRON 链）

🔒 安全与风控

安全措施：
• API 签名验证（HMAC-SHA256）- 防止伪造请求
• IP 白名单 - 限制访问来源
• 频率限制 - 防止恶意刷单（普通用户 100次/分钟）
• 私钥加密存储 - 热钱包私钥使用 AES-256 加密
• 操作日志审计 - 所有操作可追溯
• SQL 注入/XSS 防护 - 基础安全防护

风控策略：
• 订单金额限制 - 单笔最大 100 万能量，防止资源耗尽
• 用户余额监控 - 余额不足自动拒单
• 资源池预警 - TRX 可用量 < 20% 自动告警
• 异常订单检测 - 同地址短时间大量订单自动拦截
• 黑名单机制 - 恶意用户自动封禁

资金安全：
• 冷热钱包分离 - 80% TRX 存冷钱包，20% 热钱包运营
• 多签钱包 - 大额转账需要 2/3 多签
• 每日提现限额 - 单用户最多提现 5000 TRX/天
• 人工审核 - 大额提现（>1000 TRX）人工复核
